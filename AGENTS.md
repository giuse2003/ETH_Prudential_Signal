# Istruzioni per agenti - ETH Prudential Signal

## Cartella di lavoro obbligatoria

Prima di leggere, modificare, testare, aggiornare o pubblicare il progetto,
confermare che la cartella di lavoro sia:

`D:\GitHub\ETH_Prudential_Signal`

Non usare cartelle omonime, copie locali, cartelle in `Documents`, Dropbox o
altri percorsi. Se la cartella corrente non e' quella indicata sopra,
interrompere il lavoro operativo e spostarsi prima nel percorso corretto.

## Lettura prioritaria

Prima di analizzare, modificare o proporre cambiamenti al progetto, leggere in
questo ordine:

1. `DOCUMENTATION/MODEL_DOCUMENTATION_INDEX.md`
2. `DOCUMENTATION/DECISION_LOG.md`
3. `DOCUMENTATION/ETH_MODEL_RESEARCH_DIARY.md`
4. `reports/official_baseline_implementation.md`
5. `README.md`

## Strategia ufficiale

La strategia ufficiale corrente deve essere ricavata dai file sopra, non dedotta
da test sperimentali o report isolati.

In particolare:

- `DECISION_LOG.md` contiene le decisioni operative approvate.
- `ETH_MODEL_RESEARCH_DIARY.md` contiene il diario completo delle analisi.
- `MODEL_DOCUMENTATION_INDEX.md` spiega dove si trovano stato ufficiale,
  decisioni, report e roadmap.
- `reports/official_baseline_implementation.md` documenta la promozione della
  Baseline ufficiale.
- `README.md` descrive le regole operative correnti esposte pubblicamente.

## Regola fondamentale

Non modificare la Baseline ufficiale senza una richiesta esplicita.

Ogni proposta di modifica strategica deve distinguere chiaramente tra:

- strategia ufficiale;
- esperimenti storici;
- varianti candidate;
- modifiche operative da implementare.

I report in `Test_Reports/` sono output di test e non sono automaticamente
strategia ufficiale.
